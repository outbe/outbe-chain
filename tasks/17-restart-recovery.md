# T17 — Restart matrix, crash recovery, parent-root verification

## Summary

Implement the §13.3 restart matrix: startup classification of the CE marker vs the durable finalized Reth
checkpoint, event/receipt-based rebuild for the allowed lag window, fail-closed halt/resync for impossible
or corrupt states, and the pre-serve parent-root verification.

## Context

Allowed post-crash windows are asymmetric: `Reth = H, SMT = H-1` → rebuild SMT H from durable receipts,
verify root, commit, ACK; `Reth = H, SMT = H` → verify equal marker, idempotent no-op, ACK.

The behind-row rebuild path is NOT limited to the one-block ACK window (now stated normatively in amended
§13.2 — spec amendment #5): during snapshot bootstrap (T22,
import at `H` + replay `H+1..head`) an arbitrary contiguous lag `Reth = head, SMT = J` is legitimate, and a
crash mid-replay restarts into this same row — the SMT `last_applied` marker IS the durable progress
cursor, so catch-up resumes from `J+1` without any separate bootstrap staging state. Precondition: the
canonical event/receipt tail back to `J` is retained (guaranteed by §14.5's bootstrap-capability rule for
the snapshot in use). While behind, the pre-serve parent-root gate already prevents proposing/validating at
head; proof serving stays at `proof_ready_height = J` per §10.4 (clients requesting current state reject).
`Reth = H-1, SMT = H` is forbidden by the barrier → fail closed, no propose/validate/proof-serve, resync.
The rebuild path covers FINALIZED heights only; a lost speculative batch for an UNFINALIZED candidate is
regenerated by re-executing that candidate (T12, postfix PF-H10) — it is not a restart-matrix row.
Same-height hash/scheme/root conflicts and parent-mismatch gaps → halt and resync / recover from a verified
checkpoint; root mismatch at the recorded marker → never repair in place, restore from a snapshot anchored
to an independently verified finalized root. `proof_ready_height = persistent_tree_height`.

## Scope

- Startup classifier over `{tree marker, durable finalized EVM checkpoint, consensus finality}` implementing
  every §13.3 row with typed outcomes.
- Rebuild path (invoked through T16's typed `RebuildRequired` outcome — audit-final B-05; the integration
  test of the REAL handler lives here, not in T16): reconstruct the missing height's staged batch from
  durable finalized canonical core
  events/receipts (T08 decoder), apply contiguously, verify each committed root against the block's
  slot/artifact, then commit the marker (uses T15/T16). Commit and ACK are SEPARATE: verify→apply→commit
  always happens during rebuild; an ACK is NEVER emitted synthetically by the rebuild itself. The ACK for a
  recovered block is sent exactly once, in response to Marshal's real redelivery of the un-acked block
  after restart (marker already equal → idempotent no-op → ACK, per the matrix's first row). A fresh node's
  post-snapshot catch-up beyond its locally retained receipts is ordinary Marshal-driven sync (deliver →
  execute → seal → commit → ACK per block), not receipt-replay. Rebuild is domain-independent: `collection_key`/`tree_key`
  derive from the event's `{domain_id, partition_key_or_none, id_bytes}` (T08 amended shape) — no body
  decoding, no domain schema knowledge.
- Pre-serve gate: verify local parent tree root against the parent block's committed root before proposing,
  validating, or serving proofs.
- Halt/resync path: mark node degraded, refuse propose/validate/proof-serve, surface operator action
  (snapshot restore via T22 once available).
- Logging per repo security rules: block number/hash, marker heights, error class.

## Out of scope

- Snapshot import mechanics (T22); Mongo/body-store recovery (T21 — independent cursor).

## Acceptance criteria

1. Full restart-matrix test: every row reproduced by crash injection and classified correctly (§19.11).
2. Rebuild-from-receipts reproduces the exact committed root for the missing height (differential vs T04);
   multi-height contiguous catch-up test: crash injected mid-replay with `Reth = head, SMT = J` (J far
   behind) resumes from the marker and reaches head with per-height root verification.
2b. Body/event/leaf coherence during rebuild (P1-5): the rebuilder RECOMPUTES each Write event's leaf from
   the exact body/version/id fields and fail-closes on mismatch with the event's carried `leaf_value` —
   local receipt corruption is detected at the event, not only at the final root mismatch.
2c. Operator resync protocol: the halt/resync rows name the concrete operator action sequence (stop node →
   snapshot restore per T22/T28 → verify → restart) in the runbook.
3. Forbidden state (`Reth = H-1, SMT = H`) constructed artificially → node refuses all three roles.
4. Idempotent-noop row: crash between SMT commit and ACK → restart re-applies marker as no-op, then ACKs.
4b. Commit/ACK separation: rebuild of N missing heights emits ZERO ACKs by itself; each subsequent Marshal
   redelivery gets exactly one ACK (no duplicates, no synthetic ACKs — counted in the test harness).
5. Root mismatch at marker → no in-place repair attempted; degraded mode entered.

## Invariants

- Recovery is bounded and local; a single node failure cannot alter finalized network state.
- No proof is served below/around an unverified parent root.
