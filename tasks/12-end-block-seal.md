# T12 — run_end_block_seal: staged batches, Root Catalog, SealOutput

Status: todo

## Summary

Implement `CompressedEntitiesLifecycle` as the last consensus end-block module: consume the overlay, compute
staged shard/collection/catalog updates and `R_sealed(B)`, write-and-clean through one buffered post-block
hook, notify the state-root task, and return typed `SealOutput` to the executor.

## Context

Seal steps 1–10 (§8.2): read touched entities/collections → decode pending words (reject `Untouched` — a
touched entry decoding to word `0` is corruption and aborts the block; it is NOT covered by the
invalid-range check — and reject invalid encodings), map Deleted→ZERO, sort by
`{collection_key, shard_index, tree_key}`, group by collection/shard → staged
`update_all` per touched shard against parent tree/ancestor overlays (no persistent side effects) →
recompute touched collection tops and `R_collection`; ONLY retired collections lose their catalog leaf —
a collection emptied by ordinary deletes remains a *changed* collection and keeps its leaf with a
ZERO-top `R_collection` (pinned rule, see T04); update changed `collection_key → R_collection`, derive
`R_sealed(B)` → proposer exports root to artifact builder /
validator requires equality with the supplied tag-0x08 artifact → buffered hook writes
`last_sealed_root`, zeroes every pending entry and every touched-vector ELEMENT first, then finally both
vector LENGTHS (§8.2 step-7 order — resetting lengths first would strand non-zero element slots) →
post-condition assert → flush once →
`OnStateHook(StateChangeSource::PostBlock(StateChangePostBlockSource::Other("compressed_entities_seal")))` →
return `SealOutput`. Executor retains it and publishes the batch to the speculative cache keyed by block
hash after sealing; finish/sealing error drops it. Speculative batches (§13.1): immutable, in-memory, keyed
by block hash, ancestor-chain reads, bounded retention (numeric bounds Q11).

## Scope

- The lifecycle module + seal algorithm above; parent-view read path over persisted tree + staged ancestors.
- Buffered post-block storage write pattern (precedent: zerofee flush + `on_state`,
  `crates/blockchain/evm/src/executor.rs:2761-2820`); exactly one flush.
- Post-condition: both touched vectors len 0, all touched pending entries Untouched, slot 1 == computed root,
  only persistent CE EVM change is slot 1; violation aborts block.
- Speculative cache: publish/lookup/discard by block hash; losing-branch discard; rebuildability hook for T17.
- Batch envelope (audit-final H-05): every published speculative batch is self-describing —
  `{block_hash, parent_block_hash, parent_root, new_root}` — so T16 can validate at commit time that the
  batch, the durable slot-1 root, the marker, and the artifact all describe the same result; a mismatching
  envelope is unrepresentable in the cache API.
- Ancestor-chain discipline (audit-final H-08): parent-view composition over staged ancestors validates
  chain CONTINUITY (each ancestor's `block_hash`/`new_root` links to the next `parent_block_hash`/
  `parent_root`; no gaps, no fork substitution); the DIRECT parent chain of any block being built or
  validated is PROTECTED from cache eviction for the candidate window; a missing unfinalized ancestor is a
  LOCAL typed outcome (builder defers / validator abstains) — never a consensus-invalid verdict.
  Reconstruction owner (postfix PF-H10): a speculative batch lost to restart or eviction for an
  UNFINALIZED candidate is regenerated by RE-EXECUTING that candidate block against its finalized parent
  state (the normal validation path) — never by trusting a peer's batch; if the re-execution inputs are
  unavailable, the local defer/abstain outcome stands.
- Executor wiring as the last end-block module before final state-root calculation, hard-fork-governed order.

## Out of scope

- Artifact codec (T13), persistence to MDBX (T15/T16), numeric retention bounds (T24).

## Acceptance criteria

1. Differential: seal over randomized overlay states equals T04 reference `R_sealed`, incl. retirement and
   multi-collection blocks.
2. Same-block mint→delete nets to ZERO update on absent key without error; empty-touched block reproduces
   parent root without touching shard state; a collection emptied by deletes keeps a ZERO-top catalog leaf
   (differential vs T04 pinned rule) while a retired one loses it.
3. Abort paths: touched entry decoding to `Untouched`, invalid pending word, ZERO-over-content, root
   mismatch (validator), flush error — block aborted, staged batch discarded, no partial state (§19.6).
4. `SealOutput` handoff/drop paths tested: executor-finish failure drops batch; success publishes keyed by
   block hash (§19.14 item).
5. Proposer/validator equality test over identical inputs (§19.4 foundation).
6. T31 bounds enforced here with SPLIT semantics (audit v6 P0-3): max staged-tree bytes is a PROTOCOL
   bound — conservatively pre-reserved per mutation, a seal-time breach is a deterministic block failure
   (estimator bug/corrupt proposal); speculative-cache count/bytes is NODE-LOCAL — saturation triggers
   eviction/drop/rebuild (batches are reconstructible cache) or a local liveness status, and NEVER makes
   a correct block consensus-invalid; boundary tests for both at the provisional values.
6b. Staged-tree CONFORMANCE (relayered per audit v8 P0-1 — the estimator implementation lives in T10's
   `resource.rs`, formula in T31): this task computes the ACTUAL staged batch size at seal and asserts
   `actual <= reserved` as a postcondition; estimator conservatism proven by a randomized-blocks test
   that seal-time actuals never exceed the sum of reserved deltas; a breach is the deterministic block
   failure of the T31 matrix (estimator bug/corrupt proposal). The `reserved` side comes ONLY from T10's
   typed immutable block resource summary (audit-final B-01); `actual` is measured in the canonical
   SERIALIZED protocol footprint whose grammar is owned by the T30 wire spec (T31 references, T15
   implements — audit-final H-12, postfix PF-M07) — heap peak and MDBX write
   amplification are T24 report metrics, never this bound's unit.
7. Ancestor-chain tests (audit-final H-08): gap and fork-substitution fixtures rejected; eviction never
   breaks a protected direct-parent chain; a missing unfinalized ancestor yields the local defer/abstain
   outcome on both roles.
8. Envelope (audit-final H-05): every published batch carries the envelope; T16's commit-time binding
   check has a red test against a stale/wrong-fork batch.

## Invariants

- Preparation has no persistent side effects; a partially sealed block is never produced or accepted.
- Seal is the last mutation-capable lifecycle step; no later step may create a compressed mutation.
